Robot Framework - Keyword Calls

Keyword calls are highlighted in the example test case file on the right. They are the main building blocks of test cases. Typically they are written in Title Case and imperative form ("Connect To Device"). They are used to execute code that can be defined either in Robot Framework itself or in external libraries with Python.

When a keyword is called, Robot Framework looks for its definition from the various places where the keyword can be defined - the file itself, custom libraries, imported libraries or separate resource files. If it is not found anywhere, it raises an error.

Keywords may take arguments and return values. They are similar to functions in proper programming languages. 

*** Settings ***
Library    CustomLibrary.py

*** Variables ***
${SERIAL_PORT}    1234

*** Test Cases ***
Test LED Timer
    [Setup]   Connect To Device   ${SERIAL_PORT}
    Turn On LED
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    ON
    Sleep    5s
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    OFF
    [Teardown]    Disconnect From All Devices
