Robot Framework - General syntax

Similar to python and unlike to other programming languages, Robot Framework has syntactically significant line changes, indentation, and spaces.



One logical line in Robot Framework can be made up of multiple actual lines by using the line continuation character .... This is often used to make long keyword calls more readable, or to split a long list of arguments into multiple lines. A logical line ends with an end-of-line token, which is typically a newline character, unless the next line starts with the line continuation token ....

Indentation is used to determine which lines are part of a test case or keyword, and which lines start the definition of a new test case or keyword. The first line of the test case name is not indented and determines its name. All subsequent lines that are indented are considered part of that test case until another non-indented line is encountered, or a new section is called. The same applies to keyword definitions.

Any two entities (variables, keywords, control structures) are separated with a separator token. Valid separation tokens include at least TWO (2) consecutive spaces or a pipe (|). The convention generally used is FOUR (4) spaces. The same separation is also used for indentation.

Most special characters hold no significance in syntax. Single spaces are allowed in variable, keyword and test case names. However, when giving string arguments that are processed by a custom python keyword, care must be taken to ensure that the variables are read correctly. Backslashes \ can be used to escape special characters.

Library    CustomLibrary.py

*** Variables ***
${SERIAL_PORT}    1234

*** Test Cases ***
Test LED Timer
    [Setup]   Connect To Device    ${SERIAL_PORT}
    Turn On LED
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    ON
    Sleep    5s
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    OFF
    [Teardown]    Disconnect From All Devices



from robot.api.deco import library, keyword
import serial

@library
class CustomLibrary:
    def __init__(self):
        self.connection = None

    @keyword
    def connect_to_device(self, serial_port, baudrate=9600, timeout=2):
        self.connection = serial.Serial(
            port=serial_port,
            baudrate=baudrate,
            timeout=timeout
        )
