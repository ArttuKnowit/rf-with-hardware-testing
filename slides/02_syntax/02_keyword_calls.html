Robot Framework - Keyword Calls

Keyword calls are highlighted in yellow. They are the main building blocks of test cases. They are used to execute code that can be defined either in Robot Framework itself or in libraries with Python. Typically they are written in Title Case and imperative form: Connect To Device.

Broadly speaking, keywords are either user keywords or library keywords. From Robot Framework's point of view, user keywords are composite keywords meaning that they are comprised of other keyword calls, while library keywords are atomic and do not split into smaller entities from the Robot Framework point of view. However, when looking at their pythonic definitions, they can call other functions and further adaptation layers as they please, and in that sense the atomic nature clearly only applies Robot Framework-wise.

When Robot Framework is parsing the definition layer, keyword definitions are searched from the various places where the keyword can be defined - the file itself, custom libraries, imported libraries or separate resource files. If it is not found anywhere, it raises an error. You may check the definition of "Connect To Device" keyword in the CustomLibrary.py file to see how it is defined and what it does.

Keywords may take arguments and return values. They are similar to functions in proper programming languages. In some cases they ARE functions.

    [Teardown]    Disconnect From All Devices

Disconnect From All Devices
    ${devices}=    Get All Connected Devices
    FOR    ${device}    IN    @{devices}
        Disconnect From Device    ${device}
    END

CustomLibrary.py 
from robot.api.deco import library, keyword
import serial

@library
class CustomLibrary:
    def __init__(self):
        self.connection = None

    @keyword
    def connect_to_device(self, serial_port,
                          baudrate=9600, timeout=2):
        self.connection = serial.Serial(
            port=serial_port,
            baudrate=baudrate,
            timeout=timeout
        )
