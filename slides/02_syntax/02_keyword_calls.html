Robot Framework - Keyword Calls

Keyword calls are highlighted in the example test case file on the right. They are the main building blocks of test cases. They are used to execute code that can be defined either in Robot Framework itself or in libraries with Python. Typically they are written in Title Case and imperative form: Connect To Device.

Broadly speaking, keywords are either user keywords or library keywords. From Robot Framework's point of view, user keywords are composite keywords meaning that they are comprised of other keyword calls, while library keywords are atomic and do not split into smaller entities from the Robot Framework point of view. However, when looking at their pythonic definitions, they can call other functions and further adaptation layers as they please, and in that sense the atomic nature clearly only applies Robot Framework-wise.

When a keyword is being parsed as Robot Framework is executing, keyword definitions are searched from the various places where the keyword can be defined - the file itself, custom libraries, imported libraries or separate resource files. If it is not found anywhere, it raises an error.

Keywords may take arguments and return values. They are similar to functions in proper programming languages. 

*** Settings ***
Library    CustomLibrary.py

*** Variables ***
${SERIAL_PORT}    1234

*** Test Cases ***
Test LED Timer
    [Setup]   Connect To Device    ${SERIAL_PORT}
    Turn On LED
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    ON
    Sleep    5s
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    OFF
    [Teardown]    Disconnect From All Devices

*** Keywords ***
Disconnect From All Devices
    ${devices}=    Get All Connected Devices
    FOR    ${device}    IN    @{devices}
        Disconnect From Device    ${device}
    END
