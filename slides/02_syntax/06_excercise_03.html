Exercise 03 - Testing the Arduino board

  1. Using the instructions in the previous exercise, flash the Arduino with the firmware from project_02_custom directory.
  2. Run tests using either suites/project_02/part1.robot or suites/project_02/part2.robot as target with the robot command uv run robot -d results [target]. The tests should fail because the ${PORT} variable is not set correctly.
  3. Set the ${PORT} variable to the correct port for your Arduino using either a command-line variable or the *** Variables *** section in the test suite and re-run the test suite.
  4. Try running the suites separately, or together using the directory suites/project_02/ as the argument for the robot run command. The suite should now pass all tests for part 1 and 3/4 tests for part 2.
  5. Explore the logs to see why the one test fails. Is it the fault of the SUT or the tests?
