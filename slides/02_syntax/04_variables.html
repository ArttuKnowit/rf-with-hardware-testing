Robot Framework - Variables

Variables are used just like in any programming language, to store data that can be used later in the test cases. They have a prefix character (usually $) and are enclosed in curly braces {}.

They can be defined in various places and scopes. Variables specifically defined in the definition layer are considered static variables, and variables defined during run time through keyword calls and returns are considered dynamic variables. Static variables are defined at a higher scope and can be accessed from multiple test cases. Dynamic variables are defined within a specific scope (e.g., test case level) and can only be accessed within that scope. Typically static variables are written in CAPITAL LETTERS and dynamic variables in lower case.

In this example, ${SERIAL_PORT} is defined in the Variables section and has suite scope, while ${led_status} is defined dynamically in the test case and has test case scope. Variables can also be defined using the -v VARIABLE_NAME:VALUE flag on the command line, in external variable or resource files, or dynamically within custom libraries. 

*** Settings ***
Library    CustomLibrary.py

*** Variables ***
${SERIAL_PORT}    1234

*** Test Cases ***
Test LED Timer
    [Setup]   Connect To Device   ${SERIAL_PORT}
    Turn On LED
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    ON
    Sleep    5s
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    OFF
    [Teardown]    Disconnect From All Devices
