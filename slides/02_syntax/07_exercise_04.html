Exercise 04 - User Input And Custom Library

Robot Framework has the Dialogue library, that is widely used for getting user input, but it is generally not a good choice for simple systems where extra dependencies are not desired and command-line input is preferred. In this exercise, we will build our own library for getting user input from the command line, and use it to control the flow of our tests.

  1. Create a new suite file under project_02 called "part3.robot".
  2. Copy the contents of either part1.robot or part2.robot to part3.robot.
  3. Change the test cases in part3.robot to use keyword Confirm User Has Pressed Button instead of Write Serial | PressButton
  4. Create a new python file under libraries/ called OperatorLib.py
  5. Import the keyword and library decorators from robot.api.deco
  6. Define a new class called OperatorLib and decorate it with @library
  7. Define a new method in the class called confirm_user_has_pressed_button that takes no arguments and returns nothing. Decorate the method with @keyword. Use the python's native input() function to get user input.
  8. Import the library to part3.robot and run the test suite. The tests should now wait for user input instead of writing to the serial port.
  9. The dialogue on when the tests are waiting for the button push is lacking. Figure out a way to add logging to the console to inform the user when to press the button.
    • There are several ways to do this, but some are clearly better than others.

OperatorLib.py 
from robot.api.deco import library, keyword

@library
class OperatorLib:
    @keyword
    def confirm_user_has_pressed_button(self):
        logger.info("Press the button on the Arduino and then press Enter.", console=True)
        input("Press Enter after pressing the button on the Arduino.")
