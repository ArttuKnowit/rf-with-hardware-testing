Generic Test Automation Architecture

The ISTQB (International Software Testing Qualifications Board) defines a generic test automation setup by dividing it into layers.

  • Definition Layer which defines the test cases and test data.
    In the case of Robot Framework, this is the test case files and resource files, as well as any data files used in testing.
  • Execution Layer which executes the tests and collects results. This means Robot Framework, being the tool that executes the tests, falls into this layer. Note that the individual .robot files are part of the definition layer, and Robot Framework as an executable tool and runnable program is part of the execution layer.
  • Adaptation Layer which adapts the test automation framework to different environments and technologies. This is whatever tools and alyers are used between the execution layer and system under test to connect them properly. In this course, we will use Python, but it can be argued that the python serial library, the USB cord connecting the device to the machine running tests and any other software used in connecting the execution layer and SUT are part of this layer.
  • System Under Test (SUT) which is the application or system being tested. This includes both the hardware and firmware of the device that is being tested, or in case of software being under test, any part of the software system that is required to run it.
