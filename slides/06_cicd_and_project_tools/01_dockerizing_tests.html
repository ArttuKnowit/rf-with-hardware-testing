Dockerizing Robot tests

In case you are familiar with Docker, it will not come as a surprise that Robot Framework tests can be containerised with it. Containerisation can be a great way to ensure that the tests are run in a consistent environment and that all the dependencies are properly managed. It also allows running tests in an environment that is as close to production as possible, which can help to catch issues that might not be visible in a local development environment.

Each Dockerized case is unique but a good checklist for a Dockerfile for Robot Framework tests is:

  1. Ensure you use a base image that has python installed
  2. Copy all necessary files such as test suites, resources, libraries and test data, as well as requirements files for installing dependencies
  3. Install dependencies with pip (or uv if you have a Docker image with uv installed)
  4. Define the run command in the Dockerfile in a way that allows passing necessary environment variables and other run parameters
  5. Order the steps in the Dockerfile to optimize caching and build times. If you expect to rarely include new dependencies and often change the test code, place the dependency installation steps before copying the test code.
  6. Use a run command in the machine that runs the tests that gives the correct volume of results out, as well as other parameters, gives access to networks etc.


FROM python:3.12-slim

WORKDIR /app
COPY suites/ /app/suites
COPY resources/ /app/resources
COPY libraries/ /app/libraries
COPY data/ /app/data
COPY requirements.txt /app/requirements.txt

RUN pip install --no-cache-dir -r requirements.txt

CMD ["robot", "--outputdir", "/app/results", "-v", "PORT:/dev/ttyACM0", "/app/suites"]
Run with command 
docker build -t robot-tests .
docker run --rm --device /dev/ttyACM0:/dev/ttyACM0 -v $(pwd)/results:/app/results robot-tests
