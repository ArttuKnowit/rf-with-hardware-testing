Arguments in Python Files

As python is used heavily, it is important to consider how passing arguments to python is generally done. In Robot Framework, all arguments are string-first, and it is generally not possible to create variables outside basic data types (strings, integers, floats, booleans) or structures (scalars, lists, dictionaries) in Robot Framework itself. However, when calling for keywords defined in python, it is possible to pass more complex value types to arguments.

The example test case on the right will have two arguments that are inherently strings, but really represent numbers. 


*** Test Cases ***
Example Test Case 1
    VAR    ${arg1}    1
    VAR    ${arg2}    2
    Should Not Be Equal As Numbers    ${arg2}   ${arg1}    # Works because keyword arguments are converted into correct type

    Log    ${arg1 + 1}    # Does not work because we are trying to add a number to a string, no type-conversion done
