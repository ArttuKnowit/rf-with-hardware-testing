Argument Types

As variables are given to keywords, they are called arguments, just like in other programming languages when an instance of a function is called with a specific value or values. Arguments can be categorised in several ways:

  • By whether they are necessary or not:
    • Mandatory arguments are arguments that must be given, or the keyword will fail.
    • Optional arguments are arguments that are not required because they have an existing default value.
  • By their kind:
    • Positional arguments are arguments that are assigned based on their position in the argument list.
    • Named-only arguments are arguments that are assigned based on their name, allowing them to be specified in any order.
    • Variable number of positional arguments (a monstrous word I've voted to be changed in the official syllabus) are arguments that allow a keyword to accept any number of positional arguments. Essentially the same as python's my_function(*args)

from robot.api.deco import library, keyword
import subprocess

@library
class SatelliteLib:
    @keyword
    def run_command_on_satellite(self, target_machine,
                                 *commands, cli_type="ssh", **options):
        cmd = [cli_type, target_machine] + list(commands)
        timeout = options.get("timeout", None)
        result = subprocess.run(
            cmd, capture_output=True, text=True, timeout=timeout
        )
        return result.stdout
