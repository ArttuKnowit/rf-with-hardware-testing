Variables and Arguments in Files

Lists and dictionaries are native data structures in robot framework. Both work as "normal" scalar variables by using them with the ${variable_name} syntax, but also provide more advanced operations for handling multiple values and key-value pairs.

Lists are defined using the @{variable_name} syntax and can be created using the Create List keyword. They can be accessed using index notation (e.g., ${my_list}[0]) and can be iterated over using keywords like FOR loops. Lists also support various operations such as appending, extending, and slicing. The difference between the two syntaxes is that ${list_name} is used when accessing the whole list as an object, while @{list_name} is used when unpacking the list.

Similarly, dictionaries can also be accessed as individual objects or by unpacking them. They are defined using the &{variable_name} syntax and can be created using the Create Dictionary keyword. They can be accessed using key notation (three different ways, either, ${my_dict['key']}, ${my_dict}[key] or ${my_dict.key}) and can be iterated over using keywords like FOR loops.

Robot Framework is not as flexible for handling lists and dictionaries as a programming language like python, and most functionality for this is done with helper keywords from the built-in Collections library. For more complex data handling, it is often easier to use custom libraries in python that can handle lists and dictionaries more flexibly and then call those keywords from the robot framework test cases.
