Variable Scopes

Variables exist in varying scopes determined by where and how they are created. Global variables are either specifically set global using the VAR syntax and the scope=GLOBAL flag, through the command line, or simply by being specific built-in variables that are always set global, such as ${CURDIR}.

Suite scope variables are typically set from the *** Variables *** section of a suite file, or in the *** Variables *** section of a resource file that is imported into the suite. The VAR syntax with the flag scope=SUITE can also be used to set suite scope variables. Suite scope variables are available in all test cases and keywords in the suite, but not in other suites.

Test case scope variables are typically set dynamically during test execution, either by being returned from keywords or by being set with the VAR syntax without the scope=GLOBAL or scope=SUITE flag. Test case scope variables are only available within the test case they are defined in, and not in other test cases or keywords.

Variables can also exist within keywords only in what is called local scope. These are typically set dynamically during keyword execution and are only available within the keyword they are defined in. They can be created by using the VAR syntax without any scope flag, or by being returned from other keywords called within the keyword. 

*** Variables ***
${SATELLITE}   voyager    # Global variable    

*** Test Cases ***
Test Case 1
    VAR    ${sensor}    temperature_sensor    # Variable with test scope
    VAR    ${ssh_key}    123456       scope=GLOBAL    # Variable with global scope
    Example Keyword
    Log    Local variable is ${local_var}    # This will not work because ${local_var} is only available within the keyword it is defined in, in this case Example Keyword

*** Keywords ***
Example Keyword
    VAR    ${local_var}    local value    # Variable with local scope
    Another Keyword    ${local_var}
    Log    Satellite is ${SATELLITE}    # This will work as Satellite is a suite level variable
    Log    Sensor is ${sensor}    # This will work if example keyword is called by a test case or other instance where ${sensor} is defined
