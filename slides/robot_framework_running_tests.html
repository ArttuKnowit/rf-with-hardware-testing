Robot Framework - Running Tests

To run tests with Robot Framework, you can use the command line interface. The basic command to run tests is
robot target_file_or_directory
There are many flags or command line variables that can be given to customize the test run. The most common or useful ones are:

  • --include and --exclude or shorter versions -i and -e - used to include or exclude tests with specific tags.
  • --variable or -v - used to define variables from the command line.
  • --outputdir or -d - used to specify the output directory for test results.
  • --rerunfailed or -R - used to rerun failed tests from a previous test run.
  • --test or -t - used to run specific test cases by name.
In addition, there are many other flags and options that can be used to customize the test 
~project-root-folder$ robot --include critical --variable SERIAL_PORT:1234 --outputdir results tests/
