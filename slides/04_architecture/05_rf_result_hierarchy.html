Robot Framework - Result Hierarchy

In Robot Framework, each composite or atomic element gets a status during running of the suite. The rule of thumb is that failures trickle upwards. Failing a single keyword in a single test in a single suite will fail the whole suite, and the FAIL status trickles upwards. However, other sub suites and test cases and keywords can still have the PASS status.

Other possible statuses for elements after a run are SKIP and NOT RUN. SKIP indicates intentionally skipping suites or test cases dynamically, and NOT RUN indicates a keyword never reached execution due to control structures or an earlier failure.
