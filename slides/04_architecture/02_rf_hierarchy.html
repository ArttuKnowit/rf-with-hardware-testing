Robot Framework - Architecture

Key Features


Robot Framework excels in

  • Organising test suites
  • Modularity and extensibility
  • Open-source ecosystem
  • Reporting and logging

but it does not in itself structure the test suites, design keywords, or organise files. Architecture is largely customisable and many different solutions can be used to achieve a good test suite architecture.

Atomic and composite elements


In Robot Framework, the elements that form the language are categorised in the following hierarchy:

  1. Test Suites
    • Which may have subsuites to several levels
    • Test suites are composite elements because they form of other test suites and test cases
  2. Test Cases
    • Test cases are composite elements because they are formed of multiple keywords
  3. Keywords
    • Which may call other keywords to several levels
    • Keywords may be composite or atomic elements, depending on if they call other keywords or not
  4. Robot Framework language statements
    • Which are atomic elements because they cannot be further decomposed

This is reflected in the way test suites, test cases, and keywords are organised and executed within the framework. A test suite encompasses both the directories that contain .robot files, as well as the files themselves.
