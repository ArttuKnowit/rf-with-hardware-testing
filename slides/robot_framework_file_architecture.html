Robot Framework - File Architecture

A well thought out file architecture is crucial for maintainable test automation. Especially if python is used heavily, as it often is with complex systems under test that require rigorous keywords and functionality to properly use.

A good file architecture has a good separation of resources, libraries, variables, test data and test cases themselves. In addition, files need to be well documented and tagged.

On the right is an example of a simple file architecture for a test suite. The test suites are organized in folders by functionality, and the same goes for resources. Libraries and data are separated from the resources. This is just one way of organizing files, and there are many other good ways to do it. 

arduino-device-test/
├── resources/
│   ├── device_connection.resource
│   ├── temperature_sensor.resource
│   └── led_control.resource
├── libraries/
│   ├── SerialLibrary.py
│   └── BreadboardUtils.py
├── data/
│   ├── constants.py
│   └── LED_DDT_Testdata.csv
└── tests/
    ├── temperature_sensor_tests.robot
    └── LED_tests/
        ├── LED_control_tests.robot
        └── LED_data_driven_tests.robot
