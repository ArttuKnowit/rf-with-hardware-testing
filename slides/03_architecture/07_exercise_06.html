Exercise 06 - Custom reporting

Robot Framework's own reporting is considered by many to be quite bare-bones or lacking. While Robot Framework does handle reporting, the key artifact, output.xml containing all run information, is more machine readable than human readable. The log.html and report.html are older implementations of test reporting and may not meet modern expectations for test reports.

One useful tool for improving the reporting capabilities of Robot framework is the Robot Framework Dashboard. It is still a simple way of reporting results, but already provides more functionality with analysis over several test scenarios and test runs, and a more modern look. It is also quite easy to set up and use.

  1. Install the Robot Framework Dashboard using uv add robotframework-dashboard
  2. Go to part3.robot and in the settings section, assign a tag interactive for the suite using the Test Tags setting in the *** Settings *** section.
  3. Run the whole root suite suites/project_02/ with the following command: uv run robot -d results -v PORT:YOUR_PORT_NAME --skip interactive --timestampoutputs suites/project_02/
  4. Take one of the LED's, place it in the wrong way (swap anode and cathode), and run all the tests again with the same command.
  5. Disconnect the whole device and run the tests again.
  6. Fix everything (connect device, orient all LED's correctly) and run the tests again.
  7. There is now much data on several runs. Let's look at the dashboard for how the results are displayed. Run the command uv run robotdashboard robot_dashboard.html -f results
  8. Open the newly generated robot_dashboard.html in a web browser to view the dashboard.
