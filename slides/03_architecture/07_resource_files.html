Resource files

Resource files provide a Robot Framework native method of separating tests and keywords. Their main purpose is to allow organising keywords into resource files that can be imported to test suites where needed. A resource file is a file written in Robot Framework syntax, with a file extension .resource that can contain anything that a suite file can except for the *** Test Cases *** or *** Tasks *** sections.

It is good practise to store most if not all user keywords in resource files, so they can be more easily imported and reused across different files. Variables can also be defined in resource files, but it is best used only in the case the same variable is useful across many different keywords in that resource file, or if for example wanting to define a variable with a lower priority in the resource file with the intention of over-writing it through the command line.
