Exercise 05 - Better Tests

So far our LED tests on project 02 have been relying on the device itself keeping track of which LED is lit up and reading that information through serial. A more rigid way of confirming that the LED is emitting light, is to measure the voltage across the LED.

  1. Flash the Arduino with the new firmware found in firmware/project_02_wired_tests by compiling it first and then uploading it to the device.
  2. Attach three wires in the following way:
    1. One wire that connects from A2 analogue pin to the bread board, in a position between the cathode of the green LED and the resistor.
    2. One wire that connects from A1 analogue pin to the bread board, in a position between the cathode of the red LED and the resistor.
    3. One wire that connects from A0 analogue pin to the bread board, in a position between the cathode of the blue LED and the resistor.
  3. Add a new keyword to the ArduinoLib.py that measures the voltage from a specific analogue pin. If you are unsure on how to do this, check the code block "read_voltage" on the right.
  4. Create a new suite file called "part4.robot" and use the previous files as a template.
    • Change the assertion in the tests to read the voltage from the bread board instead of relying on the device's read state.
    • You may still run the tests either with physical button presses and waiting for confirmation, or by relying on the serial command for invoking the button press function.
    • Explore sufficient values. The Arduino doesn't give out more than 5V so that should be the maximum value to expect. The voltage drop across the LED isn't significant.

read_voltage 

    @keyword
    def read_analog_voltage(self, pin_index):
        self.write_serial(f"ReadAnalog {pin_index}")
        response = self.read_serial()
        voltage = float(response)
        return voltage
