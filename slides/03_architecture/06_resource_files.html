Resource files

Resource files provide a Robot Framework native method of separating tests and keywords. Their main purpose is to allow organising keywords into resource files that can be imported to test suites where needed. A resource file is a file written in Robot Framework syntax, with a file extension .resource that can contain anything that a suite file can except for the *** Test Cases *** or *** Tasks *** sections.

CustomLibrary.py 
from robot.api.deco import library, keyword
import serial

@library
class CustomLibrary:
    def __init__(self):
        self.connection = None

    @keyword
    def connect_to_device(self, serial_port,
                          baudrate=9600, timeout=2):
        self.connection = serial.Serial(
            port=serial_port,
            baudrate=baudrate,
            timeout=timeout
        )
