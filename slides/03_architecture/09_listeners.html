Listeners

Listeners are a tool the interact with Robot Framework during test execution, and affecting the execution itself. Listeners use Robot Framework's internal API to get data from the execution, such as whether a test passed or failed, what keywords were executed, and more. It allows pythonic code to be executed based on the execution of the test suite.

Listeners are useful for example for logging, debugging, and reporting purposes. They can be used to log additional information during test execution, or to create custom reports based on the execution data. They can also be used to affect the execution itself, for example by skipping certain tests based on certain conditions, or by modifying the test data during execution.

Listeners are implemented as python classes that inherit from the robot.api.interfaces modules ListenerV2 or ListenerV3. They have built-in methods such as start_test or end_keyword which allow coding specific actions or scripts to be executed whenever these events are encoutnered and executed by Robot Framework.

ArduinoListener.py 
from robot import result, running
from robot.api.interfaces import ListenerV3

class ArduinoListener(ListenerV3):

    def start_test(self, data: running.Test, result: result.Test):
        print(f"Test '{data.name}' starting.")

    def end_keyword(self, data: running.Keyword, result: result.Keyword):
        print(f"Keyword '{data.name}' ended with status {result.status}.")
