Project exercise

At this point it is a good idea to practise what we have learned in more detail.

  1. Choose another project from the Arduino projects book, or figure out a project of your own, and build it.
    • Maybe don't choose project 14 as it has a very different configuration to how it works and runs. Unless you really want to.
  2. If you choose your own project, prepare to write the firmware for it yourself (or with guidance from AI). If you choose one of the projects from the book, the firmware already exists in the firmware folder. However you may still need to modify it to allow for tests to run.
  3. Write tests for your project using Robot Framework and the libraries provided. Enhance them as needed to cover all functionality.
  4. Work with your partner. You need to use the same hardware, and likely the same firmware, but you might do different tests and later try to combine them.
  5. Lean on the Robot Framework user guide and the available documentation in this repository, as well as the document outlining how Robot Framework is used at ICEYE for any decisions on how to proceed. You may ask the instructor for any help.
  6. Aim for a test suite with up to five different test cases. Consider adding analogue pin readings for testing voltages rather than testing methods. You may need to look into the firmware to see what is available.
  7. Feel free to ask the instructor for guidance or clarification at any point.

We will spend the rest of the day on this, so work on getting as much practise on the things covered so far.
