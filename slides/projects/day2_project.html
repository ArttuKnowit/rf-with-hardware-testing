Project exercise - day 2

Let's continue on the projects we started yesterday.

  • Consider what your project does. What are the main features and functionalities? Write them down in a documentation file.
  • Consider your document as a product description. What features and functionalities should you test to ensure it works?
  • Consider what you should test. Write test cases for there features and functionalities, either in pseudo code first and then Robot Framework, or straight as Robot Framework.
  • Provide the necessary functionality behind the keywords that you have for your test cases using user and library keywords.
  • Make several test cases that run and pass/fail as expected.
  • Try to Dockerize your tests.
  • If you have any manual steps in your tests, consider what it would take to automate them.
  • Make sure your code is clean by using robotidy and robocop.
  • Put test data in a .yaml file, and create a parser in python to read it. Declare global variables from the data to use in your test cases or otherwise import the test data into your tests.
