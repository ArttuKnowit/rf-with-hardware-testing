Test Case and Keyword Settings

The test case and keyword settings can be defined within the body of test cases and keywords, respectively. They both have the following settings available:

  • [Documentation] for providing a description of the test case or keyword
  • [Setup] for specifying actions to be performed before the test case or keyword
  • [Teardown] for specifying actions to be performed after the test case or keyword
  • [Tags] for assigning tags to the test case or keyword
  • [Timeout] for setting a timeout for the test case or keyword

In addition, keywords have the [Arguments] setting for defining the arguments that the keyword takes. The [Return] setting used to exist for keywords but is deprecated as it was replaced the RETURN statement. 


*** Test Cases ***
Test LED Timer
    [Documentation]    Led Timer Test
    ...
    ...    Connects to device, turns LED on and off, and in between checks for correct statuses.
    [Setup]   Connect To Device    ${SERIAL_PORT}
    [Tags]    critical    led_control
    Turn On LED
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    ON
    Sleep    5s
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    OFF
    [Teardown]    Disconnect From All Devices


*** Keywords ***
Get LED Status
    [Documentation]    Return the status of an LED with a given pin number.
    ...
    ...    Takes the pin number as an argument and returns 
    ...    True if the voltage is high and False if it is low.
    [Arguments]    ${led_pin}
    ${voltage}=    Get Pin Voltage    ${led_pin}
    ${status}=    Set Variable If    '${voltage}' == 'HI'    ${True}    ${False}
    RETURN    ${status}
