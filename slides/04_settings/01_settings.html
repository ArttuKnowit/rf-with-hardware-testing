Robot Framework - Settings

Robot Framework settings happen on each level of hierarchy: Suite, Test Case and Keyword. Settings are used to define the resources, libraries, variables, tags, and other configurations that are needed for the execution of the test cases and keywords.

Suite settings are defined in a specific section of the .robot file, which is denoted by the header *** Settings ***. Test Case and Keyword level settings are defined within the bodies of each, enclosing each setting in brackets as seen on the example. Some settings are unique to their level.

Each setting has its own syntax and usage, and they can be used to control various aspects of the test execution, such as setup and teardown, test case tags, library imports, variable definitions, and more.

*** Settings ***
Library    CustomLibrary.py
Resource    arduino_led_control.resource
Variables    variables.py
Suite Setup    Connect To Device    ${SERIAL_PORT}
Suite Teardown    Disconnect From All Devices


*** Test Cases ***
Test LED Timer
    [Documentation]    Led Timer Test
    ...
    ...    Connects to device, turns LED on and off, and in between checks for correct statuses.
    [Setup]   Connect To Device    ${SERIAL_PORT}
    Turn On LED
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    ON
    Sleep    5s
    ${led_status}=    Get LED Status
    Should Be Equal    ${led_status}    OFF
    [Teardown]    Disconnect From All Devices


*** Keywords ***
Get LED Status
    [Documentation]    Return the status of an LED with a given pin number.
    ...
    ...    Takes the pin number as an argument and returns 
    ...    True if the voltage is high and False if it is low.
    [Arguments]    ${led_pin}
    ${voltage}=    Get Pin Voltage    ${led_pin}
    ${status}=    Set Variable If    '${voltage}' == 'HI'    ${True}    ${False}
    RETURN    ${status}
